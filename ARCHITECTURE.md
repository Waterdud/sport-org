## 🔄 Поток данных

### 1. Создание события:

```
Пользователь → create_event.php
                    ↓
            Валидация формы
                    ↓
            Проверка авторизации
                    ↓
        INSERT INTO events (...)
                    ↓
        INSERT INTO participants
         (создатель = участник)
                    ↓
        Триггер обновляет счётчик
                    ↓
        Редирект → event.php?id=X
```

### 2. Присоединение к событию (AJAX):

```
Кнопка "Участвовать" → ajax/join_event.php
                             ↓
                    Проверка CSRF токена
                             ↓
                    Проверка авторизации
                             ↓
                  Проверка свободных мест
                             ↓
              INSERT INTO participants
                             ↓
              Триггер обновляет счётчик
                             ↓
          INSERT INTO notifications (организатору)
                             ↓
              JSON ответ → JavaScript
                             ↓
              Обновление UI без перезагрузки
```

### 3. Добавление комментария (AJAX):

```
Форма комментария → ajax/add_comment.php
                            ↓
                    Проверка CSRF токена
                            ↓
                    Проверка авторизации
                            ↓
                Валидация текста комментария
                            ↓
              INSERT INTO comments
                            ↓
          INSERT INTO notifications
                            ↓
          JSON с HTML разметкой → JavaScript
                            ↓
          Добавление в DOM без перезагрузки
```

### 4. Система уведомлений:

```
Событие (участие/комментарий/оценка)
                ↓
    INSERT INTO notifications
                ↓
    header.php получает COUNT(*) WHERE is_read=0
                ↓
    Отображение badge с числом
                ↓
    Клик → notifications.php
                ↓
    Список всех уведомлений
                ↓
    Клик "Прочитать" → AJAX mark_notification_read.php
                ↓
    UPDATE is_read=1
                ↓
    Обновление badge через get_unread_count.php
```

## 📊 Производительность

### Оптимизация запросов:

```sql
-- Индексы для быстрого поиска
INDEX idx_events_date (event_date, event_time)
INDEX idx_events_sport (sport_type)
INDEX idx_events_location (location_id)
INDEX idx_participants_user (user_id)
INDEX idx_participants_event (event_id)

-- Представление для частых запросов
CREATE VIEW events_full AS
SELECT e.*, u.username, l.name as location_name
FROM events e
JOIN users u ON e.creator_id = u.id
JOIN locations l ON e.location_id = l.id;

-- Триггер для автоматического обновления
DELIMITER //
CREATE TRIGGER update_participant_count
AFTER INSERT ON participants
FOR EACH ROW
BEGIN
    UPDATE events 
    SET current_participants = (
        SELECT COUNT(*) FROM participants 
        WHERE event_id = NEW.event_id
    )
    WHERE id = NEW.event_id;
END//
```

### Кеширование (.htaccess):

```apache
# Кеширование статических файлов
<FilesMatch "\.(jpg|jpeg|png|gif|css|js|ico)$">
    Header set Cache-Control "max-age=2592000, public"
</FilesMatch>

# Gzip сжатие
<IfModule mod_deflate.c>
    AddOutputFilterByType DEFLATE text/html text/css text/javascript
</IfModule>
```

## 🎯 API Endpoints

## 🔄 Жизненный цикл события

```
┌─────────────────────────────────────────────────────────────┐
│  1. СОЗДАНИЕ                                                │
│     • Пользователь заполняет форму                          │
│     • Валидация данных                                      │
│     • INSERT INTO events                                    │
│     • Создатель автоматически становится участником         │
│     • Статус: active                                        │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│  2. НАБОР УЧАСТНИКОВ                                        │
│     • Другие пользователи присоединяются                    │
│     • Триггер обновляет current_participants                │
│     • Уведомления организатору                              │
│     • Комментарии                                           │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│  3. ПРОВЕДЕНИЕ                                              │
│     • Дата наступает                                        │
│     • Событие проходит                                      │
│     • Организатор может отметить посещаемость               │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│  4. ЗАВЕРШЕНИЕ                                              │
│     • Статус: completed (автоматически после даты)          │
│     • Организатор может оценить участников                  │
│     • Обновление рейтингов (триггер)                        │
│     • Обновление статистики посещаемости                    │
└──────────────────────┬──────────────────────────────────────┘
                       │
┌──────────────────────▼──────────────────────────────────────┐
│  5. ИСТОРИЯ                                                 │
│     • Событие доступно в истории                            │
│     • Комментарии сохранены                                 │
│     • Рейтинги учтены                                       │
└─────────────────────────────────────────────────────────────┘

АЛЬТЕРНАТИВНЫЙ ПУТЬ:
┌─────────────────────────────────────────────────────────────┐
│  ОТМЕНА                                                     │
│     • Организатор отменяет событие                          │
│     • Статус: cancelled                                     │
│     • Уведомления всем участникам                           │
│     • Событие остаётся в БД для истории                     │
└─────────────────────────────────────────────────────────────┘
```
